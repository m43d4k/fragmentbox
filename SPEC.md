# fragmentbox 現行仕様

## 1. 文書の位置づけ

本書は、fragmentbox の現行コード、設定、README、既存の個別仕様書およびテストから確認できる実装済み仕様をまとめたものである。

- 対象バージョン: `pyproject.toml` の `0.1.0`
- 対象コンポーネント:
  - 投稿 GUI: `fragmentbox.py`
  - Web ビューア/API: `viewer.py`、`viewer.html`、`css/viewer.css`
- 既存の `spec_post.md` と `spec_viewer.md` は要件の背景資料として扱い、本書では現行コードの挙動を優先する。
- 将来の要望ではなく、現時点で実装されている振る舞いを記載する。

## 2. 目的と構成

fragmentbox は、短いテキストや画像、URLをローカルへ素早く保存し、ブラウザで検索・閲覧・編集するためのパーソナルツールである。

システムは次の2つの独立したプロセスで構成される。

1. PySide6 製の投稿 GUI が、1件のフラグメントを1つのMarkdownファイルとしてInboxへ保存する。
2. FastAPI 製のビューアが、InboxまたはArchiveのMarkdownファイルを読み取り、Vanilla JavaScript製のSPAへJSON APIとして提供する。

両プロセスは、プロジェクトルートの `config.toml` を起動時に一度だけ読み込み、同じ保存先を共有する。設定変更を反映するには各プロセスの再起動が必要である。

## 3. 動作環境と依存関係

### 3.1 対応環境

- Python 3.12以上
- macOSまたはLinux
- Webビューアを利用するためのWebブラウザ
- URLメタデータ取得のためのネットワーク接続

### 3.2 主な依存ライブラリ

| 用途 | ライブラリ |
| --- | --- |
| GUI | PySide6 |
| 一般URLのメタデータ取得 | trafilatura |
| YouTubeのメタデータ取得 | yt-dlp |
| ラスター画像変換 | pyvips |
| Web API | FastAPI、Pydantic |
| ASGIサーバー | uvicorn |

依存バージョンの正確な条件は `pyproject.toml` を正とする。

### 3.3 エントリポイント

パッケージには次のコマンドエントリポイントが定義されている。

- `fragmentbox` → `fragmentbox:main`
- `fragmentbox-viewer` → `viewer:main`

ただし現行の `viewer.py` に `main()` 関数は存在しないため、`fragmentbox-viewer` エントリポイントは利用できない。ビューアは `uv run python viewer.py` のようにスクリプトとして起動する。

リポジトリにローカル用の起動スクリプトが存在する場合があるが、`*.sh` はGit管理対象外であり、本仕様の配布物には含めない。

## 4. 設定

設定ファイルは、各Pythonファイルと同じディレクトリにある `config.toml` である。

```toml
[paths]
inbox   = "~/idea_pool/fragmentbox/inbox"
archive = "~/idea_pool/fragmentbox/archive"
trash   = "~/idea_pool/fragmentbox/Trash"
assets  = "~/idea_pool/fragmentbox/assets"

[viewer]
port = 8765

[tags]
presets = ["idea", "todo", "ref"]

[images]
quality = 80
```

| キー | 必須 | 使用箇所 | 仕様 |
| --- | --- | --- | --- |
| `paths.inbox` | 必須 | GUI/API | 新規フラグメントの保存先、Inboxの参照・更新・削除元 |
| `paths.archive` | 必須 | API | Archiveの参照・更新先 |
| `paths.trash` | 必須 | API | Inbox削除時の移動先 |
| `paths.assets` | 必須 | GUI/API | 画像保存先および `/assets` の配信元 |
| `viewer.port` | 任意 | API | 待受ポート。省略時は `8765` |
| `tags.presets` | 任意 | GUI | タグボタン一覧。省略時は空配列 |
| `images.quality` | 任意 | GUI | WebP変換品質。省略時は `80` |

パスは `~` をユーザーのホームディレクトリへ展開する。相対パスを指定した場合は、プロセスのカレントディレクトリを基準とする。

## 5. データ仕様

### 5.1 ディレクトリ

- Inbox: 新規保存されたフラグメント
- Archive: 外部操作などでInboxから移されたフラグメント
- Assets: 添付画像とダウンロード済みサムネイル
- Trash: ビューアから削除したInboxフラグメントと、その本文から参照される画像

GUIは保存時にInboxとAssetsを必要に応じて作成する。ビューアはモジュール読込時にAssetsを作成し、削除時にTrashを作成する。InboxとArchiveは存在しなくてもよく、その場合APIは空一覧を返す。

Archiveへ移動する機能は本アプリ内にはない。

### 5.2 フラグメントファイル

- 形式: UTF-8のプレーンテキストMarkdown
- 拡張子: `.md`
- 原則のファイル名: `YYYYMMDD_HHMMSS.md`
- 1ファイルが1フラグメントに対応する。
- GUI保存時は本文の先頭・末尾の空白を除去してから書き込む。
- GUIは空白だけの本文を保存しない。
- 同一秒内に複数回保存した場合は同じファイル名になり、既存ファイルを上書きする。

ビューア上のフラグメントIDはファイル名から拡張子を除いた値である。作成日時は、IDが `YYYYMMDD_HHMMSS` 形式ならその値から生成し、それ以外ならファイルの更新日時を使用する。

### 5.3 タグ

- 本文内の `#タグ名` をタグとみなす。
- サーバー側の抽出規則はPython正規表現 `#(\w+)` である。
- ブラウザ側の表示規則はJavaScript正規表現 `/#([\p{L}\p{N}_]+)/gu` である。
- 英数字、アンダースコア、日本語を含むUnicodeの文字・数字を扱える。
- タグは本文中のどの位置にあっても抽出対象になる。
- 大文字・小文字は区別する。
- `#favorite` はお気に入り状態を表す予約的なタグである。

### 5.4 画像参照

ローカル画像は原則として次のMarkdown記法で参照する。

```markdown
![](../assets/ファイル名)␠␠
```

例中の `␠␠` は半角スペース2つを表す。画像行の末尾は、Markdownの強制改行を表す半角スペース2つと改行である。削除時に関連画像として認識されるのは、`../assets/` を参照するMarkdown画像だけである。

### 5.5 URLメタデータ

取得できた情報はURLの直後へ次の順序で挿入する。値が空の項目は挿入しない。

```markdown
https://example.com␠␠
title: ページタイトル␠␠
sitename: サイト名␠␠
description: 説明␠␠
![](../assets/サムネイル.webp)␠␠
```

例中の `␠␠` は半角スペース2つを表す。各行は半角スペース2つと改行で終わる。メタデータ値に追加のエスケープや改行の正規化は行わず、取得元が返した文字列をそのまま使用する。

## 6. 投稿GUI

### 6.1 画面

- ウィンドウタイトルは `fragmentbox`。
- 幅は520px固定、高さは内容に応じて決まる。
- 背景色は `#2b2b2b` のダークテーマ。
- 高さ160pxのテキスト入力欄、タグボタン列、ステータス表示、`IMG` ボタン、`Save` ボタンを持つ。
- 起動時はテキスト入力欄へフォーカスする。
- `Escape` でウィンドウを閉じる。

### 6.2 タグ入力

- `tags.presets` の順序で `#タグ名` ボタンを表示する。
- ボタンを押すと、入力本文の末尾へタグを追加する。
- 同じ文字列の `#タグ名` が本文中に既に含まれる場合は追加しない。
- プリセットタグが本文に1つもない場合、最初のタグは空行を1行挟む `\n\n#タグ名` として追加する。
- いずれかのプリセットタグが既に含まれる場合、次のタグは半角スペース区切りで追加する。
- 重複判定は文字列包含であり、タグ境界を厳密には判定しない。

### 6.3 画像添付

対応する入力拡張子は次のとおりで、大文字・小文字は区別しない。

`.png`、`.jpg`、`.jpeg`、`.gif`、`.webp`、`.svg`、`.bmp`、`.tiff`、`.tif`

画像は次の方法で追加できる。

- テキスト入力欄へのローカルファイルのドラッグ＆ドロップ
- `IMG` ボタンから開く複数選択可能なファイルダイアログ

ドラッグ＆ドロップは、列挙した拡張子を持つローカルファイルが1つ以上含まれる場合だけ画像ドロップとして受理する。受理したドロップに非対応ファイルが混在している場合、そのファイルは無視する。

保存処理は次のとおりである。

- SVGとGIFは変換せず、元のメタデータを保つコピーとしてAssetsへ保存する。
- その他の対応画像はEXIF等の向き情報を適用してからWebPへ変換する。
- WebP品質は `images.quality` を使用する。
- 保存名は `YYYYMMDD_HHMMSS_ランダム8桁` と、コピー時は元拡張子、変換時は `.webp` の組み合わせとする。
- 成功時は本文末尾へ画像参照を追加し、ステータスに保存名を表示する。
- 失敗時は本文を変更せず、ステータスにエラーを表示する。

### 6.4 URL検出とメタデータ取得

- 保存時、本文内の `http://` または `https://` から始まり空白までの文字列をURL候補として検出する。
- URL末尾の `. , ; : ! ? ( ) ' \" >` は検出結果から除外する。
- YouTubeの `youtube.com` と `youtu.be` はyt-dlp、それ以外はtrafilaturaで取得する。
- YouTubeではタイトル、固定サイト名 `YouTube`、チャンネル名または投稿者名、サムネイルURLを利用する。
- 一般URLではタイトル、サイト名、説明、画像URLを利用する。相対画像URLはページURLを基準に絶対URLへ変換する。
- 同じURLが複数回現れる場合も、それぞれをURL候補として検査する。
- URLがある行の後続する非空行内に `title:` で始まる行があれば、そのURLは取得済みとみなす。

メタデータ取得は `QThread` 上でURLごとに直列実行する。取得中はSaveボタン、IMGボタン、保存ショートカットを無効化するが、テキスト入力欄とタグボタンは無効化しない。取得完了後、メタデータを現在の入力内容へ挿入して保存する。

個々のURLの取得失敗はログへ記録し、そのURLのメタデータを空として残りの処理を続行する。すべてのURLで取得に失敗しても本文自体は保存する。

### 6.5 サムネイル取得

- HTTP User-Agentとして `Mozilla/5.0 (compatible; fragmentbox)` を送る。
- 接続タイムアウトは10秒。
- `Content-Type` が `image/` で始まらない応答は保存しない。
- `Content-Length` または実際の読込量が10 MiBを超える画像は保存しない。
- URLパスの拡張子が対応外の場合は、一時ファイルの拡張子として `.jpg` を使用する。
- ダウンロード後は通常の画像添付と同じ処理でAssetsへ保存する。
- ダウンロードまたは変換の失敗はログへ記録し、サムネイルなしで処理を継続する。

### 6.6 保存操作

保存はSaveボタンまたは次のショートカットで実行する。

| 環境 | ショートカット |
| --- | --- |
| macOS | `Command+S`、`Command+Return`、`Control+Return` |
| Linux | `Control+S`、`Control+Return` |

処理順は次のとおりである。

1. 先頭・末尾を除去した本文が空なら保存せず、入力要求を表示する。
2. 未取得のURLがあればバックグラウンドでメタデータを取得する。
3. 取得結果をURL直後へ挿入する。
4. Inboxを必要に応じて作成し、Markdownファイルを書き込む。
5. 成功ステータスにファイル名を表示し、入力欄をクリアする。

環境変数 `FRAGMENTBOX_LOG` でPythonのログレベルを指定できる。無効な値または未指定時は `INFO` を使用する。

## 7. WebビューアとAPI

### 7.1 サーバー

- `viewer.py` を直接実行すると `127.0.0.1` の設定ポートでuvicornを起動する。
- 起動から約0.8秒後に `http://127.0.0.1:<port>` を既定ブラウザで開く。
- 外部インターフェースにはバインドしない。
- `/` は `viewer.html`、`/css` はプロジェクトの `css/`、`/assets` は設定されたAssetsを配信する。
- 認証、認可、CSRF対策は実装しない。ローカルホスト専用での利用を前提とする。

### 7.2 Fragment JSONモデル

```json
{
  "id": "20240315_093000",
  "created_at": "2024-03-15T09:30:00",
  "content": "本文 #idea",
  "tags": ["idea"]
}
```

- `id`: ファイルのstem
- `created_at`: ファイル名またはmtimeに基づくタイムゾーン情報なしの日時
- `content`: 読込時に先頭・末尾の空白を除去した本文
- `tags`: 本文から出現順に抽出したタグ。重複は保持する。

### 7.3 API一覧

#### `GET /api/fragments?source=inbox|archive`

指定ソースの `*.md` をファイル名の降順で返す。`source` の既定値は `inbox`。ディレクトリが存在しない場合は空配列を返す。

#### `GET /api/tags?source=inbox|archive`

指定ソースの全Markdownから抽出したタグを、重複を除去して昇順で返す。`source` の既定値は `inbox`。ディレクトリが存在しない場合は空配列を返す。

#### `PATCH /api/fragments/{id}/favorite?source=inbox|archive`

- 対象本文に文字列 `#favorite` が含まれる場合は、正規表現 `[ \t]*#favorite\b` に一致する箇所をすべて除去する。
- 含まれない場合は、末尾の空白を除去した本文に ` #favorite` を追加する。
- ファイル末尾には改行を1つ付けて保存する。
- InboxとArchiveの両方を更新できる。
- 成功時は `{"status":"ok","favorited":真偽値,"tags":[...]}` を返す。
- 対象がなければ404 `Fragment not found` を返す。

#### `PUT /api/fragments/{id}?source=inbox|archive`

- JSON本文 `{"content":"更新後の本文"}` を受け取る。
- 受け取った文字列を空文字も含め、そのままUTF-8で上書きする。
- InboxとArchiveの両方を更新できる。
- 成功時は更新後のFragmentを返す。返却時の `content` は読込処理により前後の空白が除去される。
- 対象がなければ404 `Fragment not found` を返す。

#### `DELETE /api/fragments/{id}`

- Inbox内の対象MarkdownをTrashへ移動する。Archiveは削除対象にできない。
- 本文内の `![](../assets/ファイル名)` 参照を抽出し、存在する画像もTrash直下へ移動する。
- 存在しない参照画像は無視する。
- 成功時は `{"status":"moved","id":"..."}` を返す。
- 対象がなければ404 `Fragment not found` を返す。

`source` に `inbox`、`archive` 以外を指定した場合やリクエストモデルが不正な場合は、FastAPIの入力検証により422を返す。

移動先に同名ファイルがある場合などのファイルシステムエラーは個別に処理せず、サーバーエラーとなる。MarkdownをTrashへ移動した後に画像移動が失敗した場合、ロールバックは行わない。

## 8. Webフロントエンド

### 8.1 初期表示とソース切替

- 表示言語は日本語指定だが、主要なUIラベルは英語である。
- 初期ソースはInbox。
- フラグメントと使用中タグを並列に取得する。
- フラグメントを1件1カードで表示し、表示件数を `N notes` と表示する。
- APIが返す順序を保つため、通常は新しいファイル名のカードが上になる。
- Inbox/Archiveタブ切替時は、選択タグ、検索語、日付範囲、お気に入り限定を初期化して再読込する。
- AND/ORモードはソース切替時にも現在値を保持する。
- Archiveでもお気に入り変更と編集ができる。削除ボタンだけを表示しない。

### 8.2 検索とフィルタ

- テキスト検索は入力の前後を除去し、本文全体を大文字・小文字を区別せず部分一致検索する。
- 使用中タグを複数選択できる。
- ANDモードは選択タグをすべて含むカード、ORモードはいずれかを含むカードを表示する。
- 本文中の表示タグをクリックすると、そのタグを選択状態へ追加する。
- FromとToの日付は片方だけでも指定でき、境界日を含む。
- お気に入り限定ボタンは `favorite` タグを持つカードだけを表示する。
- タグと日付には個別のクリアボタンがある。
- 条件に一致しない場合は `No notes found` を表示する。

### 8.3 本文描画

- 全出力はHTMLエスケープしてから装飾する。
- 通常行のURLは新しいタブで開くリンクにする。リンクには `rel="noopener"` を付ける。
- タグをクリック可能なインライン表示にする。
- 連続する空行は表示上1つへ圧縮する。
- 行末の半角スペース2つは表示時に除去する。
- 単独行のURL直後に `title:`、`sitename:`、`description:` の連続行がある場合、リンクプレビューとして描画する。
- `title:` がない場合はリンクプレビューにせず、URLを通常リンクとして表示する。
- Markdown画像が単独行にある場合は画像を遅延読込する。`../assets/` で始まる参照は `/assets/` へ置き換える。
- 画像読込に失敗した場合は `(画像なし)` を表示する。
- 画像クリックでライトボックスを開き、ライトボックスクリックまたは `Escape` で閉じる。
- Markdown全般を解釈するレンダラーではなく、URL、3種のメタデータ行、画像、タグだけを独自に解釈する。

### 8.4 お気に入り、編集、削除

- 各カードにお気に入りボタンと編集ボタンを表示する。
- Inboxのカードにだけ削除ボタンを表示する。
- お気に入り操作の成功後は返却されたタグでカード状態を更新する。
- 編集ボタンは生の本文をtextareaへ表示する。SaveでPUTし、成功後は返却されたFragmentでカードを更新する。
- 編集はCancel、オーバーレイクリック、`Escape` で破棄できる。
- 削除は確認モーダルで承認後にDELETEし、対象カードを表示一覧から除外する。
- 削除確認はCancelまたはオーバーレイクリックで閉じる。

フロントエンドはAPIレスポンスのHTTPステータスを明示的に検査せず、通信中表示や失敗通知も実装していない。そのため更新・削除に失敗した場合、画面とファイルの状態が一致しないことがある。

## 9. 現行実装の制約

- GUIのファイル名は秒単位なので、同一秒の保存は衝突する。
- メタデータ取得中も本文を編集できるため、取得開始後の編集内容に対して、開始時点で検出した位置情報を使ってメタデータを挿入する場合がある。
- `title:` の有無だけで取得済みを判断するため、タイトルを取得できなかったURLは次回保存時に再取得される。
- 同じURLの一部の出現だけが取得済みの場合、未取得URLの出現番号と本文全体での出現番号がずれ、別の出現位置へメタデータを挿入する可能性がある。
- URL検出と表示は、括弧を含むURLや句読点を含むURLを完全には扱えない。
- タグの追加・お気に入り判定には文字列包含判定が含まれ、常に厳密なタグ境界判定ではない。
- ビューアの日時はタイムゾーン情報を持たず、ブラウザ側でローカル日時として表示される。
- APIに同時更新制御、ファイルロック、原子的な複数ファイル移動、履歴管理はない。
- Archiveへの移動、Trashからの復元、完全削除は実装されていない。
- APIはローカルホスト上でも認証を行わない。
- `fragmentbox-viewer` のパッケージコマンドは現状利用できない。

## 10. テストで確認されている範囲

`test_viewer.py` のユニットテストは、次を確認する。

- 英数字、日本語、複数タグ、`favorite` タグの抽出
- `../assets/` 画像参照の抽出と外部画像URLの除外
- FragmentのID、ファイル名からの日時、前後空白除去、タグ抽出
